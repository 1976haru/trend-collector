# 🗺 ROADMAP — 공무원 / 업무용 확장

내부 직원 MVP (현재 라운드) 이후의 확장 방향.

---

## 1단계 — MVP (현재)

| 항목 | 상태 |
|------|------|
| 단일 비밀번호 로그인 (회원가입 X) | ✅ |
| 키워드 / 제외 키워드 등록 | ✅ |
| 메일 수신자 등록 | ✅ |
| 즉시 수집 + 매일 자동 수집(KST) | ✅ |
| 보고서 PDF 출력 (인쇄 다이얼로그) | ✅ |
| SMTP 메일 발송 | ✅ |
| Render 단일 서비스 배포 | ✅ |

---

## 2단계 — 보고서 품질

- "요약 / 주요 이슈 / 시사점 / 참고 링크" 템플릿 — *기본형은 1단계에 포함*. 머리말(부서·작성자) 사용자 입력 필드 추가.
- LLM 본문 자동 요약 — Anthropic Claude 또는 OpenAI 로 1~3줄 요약 생성. API 키는 서버 환경변수.
- **PDF 파일 첨부** 메일 — Puppeteer 로 서버에서 직접 PDF 바이너리 생성.
- 매체 등급 (중앙 / 지방 / 인터넷) 분포 표 — `mediaList.js` 를 서버로 옮겨 보고서 템플릿에서 사용.
- 키워드 급상승 감지 — 직전 리포트와 비교하여 50% 이상 증가한 키워드 알림.

---

## 3단계 — 정책·행정 특화

- **부처별 키워드 프리셋** — 보건복지부 / 행정안전부 / 국토교통부 등 그룹.
- **17개 광역지자체 빠른 필터** — 지방지 매체 매핑.
- **법률·고시·공고 RSS 추가** — 법제처 국가법령정보, 정부24 공고.
- **부고 / 인사 / 동정 자동 분류**.
- 광고·홍보성 기사 필터 (현재는 단순 키워드) → 도메인 블랙리스트로 보강.

---

## 4단계 — 다중 부서 / 팀 운영

- ADMIN_PASSWORD 1개 → **팀(부서)별 별도 비밀번호** + 별도 키워드/수신자 묶음.
- 그래도 회원가입은 만들지 않고 운영자가 부서별 토큰 발급 형태로 시작.
- 발송 이력 / 열람 로그 (감사용).

---

## 5단계 — 백엔드 정식화

- **Postgres (Render Postgres or Supabase)** 로 config / reports 이전.
- 로그인 시도 / 발송 이력 영구 보관.
- Express → 필요 시 Fastify 로 교체.
- 컨테이너 (Dockerfile) + Cloud Run / Fly.io 등 멀티 클라우드 옵션.

---

## 6단계 — 분석 / AI

- 이슈 클러스터링 (제목 임베딩 → 코사인).
- 매체별 / 키워드별 보도 트렌드 시계열 차트.
- Claude Sonnet 으로 "오늘의 핵심 이슈 5개" 자동 발문.
- 정책 영향 분석 (특정 정책 키워드의 보도 추이를 다른 키워드와 교차).
