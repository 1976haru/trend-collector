# Trend Collector FAQ — 자주 묻는 질문

앱 안 **❓ 도움말** 탭 에서도 동일한 내용을 볼 수 있습니다.

---

### Q. 키워드는 몇 개 넣어야 하나요?

**A.** 처음에는 1~3개 정도만 넣는 것이 좋습니다. 너무 많으면 관련 없는 기사가 함께 들어옵니다.

---

### Q. 여러 키워드를 넣으면 모두 포함된 기사만 나오나요?

**A.** 아닙니다. 기본은 하나라도 포함되면 결과에 포함되는 **OR 검색** 입니다.

---

### Q. 모든 키워드를 포함하는 기사만 옵션은 언제 쓰나요?

**A.** 아주 좁게 검색하고 싶을 때만 사용합니다. 대부분의 경우 결과가 0건이 됩니다.

`보호관찰` 과 `보호관찰소` 처럼 포함 관계인 키워드는 자동으로 더 긴 키워드 기준으로 완화됩니다.

---

### Q. PDF가 안 되면 어떻게 하나요?

**A.** **Word** 또는 **HTML** 다운로드를 사용하면 됩니다. HTML 은 브라우저로 열고 **Ctrl+P** 로 인쇄 / PDF 저장이 가능합니다.

---

### Q. 네이버 API 는 직원마다 입력해야 하나요?

**A.** 아닙니다. 관리자가 한 번 저장하면 PC 와 스마트폰에서 공통으로 사용됩니다.

---

### Q. 테스트 메일 발송이 Connection timeout 으로 실패합니다.

**A.** Render 무료 플랜에서 SMTP 포트(25 / 465 / 587)가 차단되어 발생할 수 있습니다.
**Resend / SendGrid 같은 메일 API 방식**을 사용하거나 Render **유료 플랜**으로 전환하세요.
앱 내 **🛠 설정 → 📨 메일 설정** 에서 발송 방식을 선택할 수 있습니다.

---

### Q. 기능개선 제안은 메일 발송이 실패해도 보존되나요?

**A.** 네. 제안 내용은 항상 서버의 `data/feedback.json` 에 저장됩니다.
메일이 실패해도 관리자 화면(**🛠 설정 → 📨 기능개선 제안**)에서 모든 제안을 확인할 수 있습니다.

---

### Q. 새 PC 에서 처음 접속했는데 키워드가 자동으로 들어가 있어요.

**A.** 서버에 저장된 설정이 남아 있는 경우입니다. 키워드 화면에서 **🗑 키워드 전체 초기화** 버튼을 누르면 비울 수 있습니다.
신규 배포 / 서버 초기 상태에서는 키워드가 비어 있어야 정상입니다.

---

### Q. 같은 기관에서 여러 명이 동시에 사용해도 되나요?

**A.** 네. **키워드 / 수신자 / 메일 설정** 은 모든 직원이 공유합니다. 한 명이 바꾸면 모두에게 적용됩니다.

---

### Q. 편철형과 분석형 차이는 무엇인가요?

**A.** 편철형은 인쇄용 **언론 스크랩철**, 분석형은 보고용 **분석 자료** 입니다.
두 가지를 별도 다운로드로 제공하며, 리포트 상세 화면에서 영역이 분리되어 있습니다.

---

### Q. 보고서를 자동으로 메일로 받고 싶어요.

**A.** **🛠 설정 → 📅 자동 수집 설정** 에서 “자동 메일 발송” 토글을 켜고, **📧 수신자 탭** 에서 이메일을 등록하세요.

---

### Q. 리포트가 0건이에요. 어떻게 확인하나요?

**A.** 리포트 상세 화면 상단의 **🔬 검색 진단** 패널에서 어느 단계(원본 / 날짜 / 중복 / AND 필터)에서 결과가 제외되었는지 확인하세요.
“모든 키워드 포함” 옵션이 켜져 있다면 먼저 해제하세요.

---

### Q. 일부 기사 본문이 비어 있어요.

**A.** 일부 매체는 봇 차단 / 로그인 요구로 본문이 자동 추출되지 않을 수 있습니다.
RSS 메타데이터로 자동 대체되며, **🔄 이 기사 재추출** 또는 원문 링크에서 직접 확인 가능합니다.

---

### Q. 리포트 표지 / 날짜 / 기관명을 바꾸려면?

**A.** 리포트 상세 화면 상단의 **📰 편철 설정** 패널에서 표지 제목 / 날짜 / 중앙 박스 / 기관명을 직접 수정할 수 있습니다.
**💾 편철 설정 저장** 후 PDF / Word / HTML 다운로드에 모두 반영됩니다.
