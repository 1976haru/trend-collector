# Trend Collector 사용자 가이드

처음 사용하는 분도 따라 할 수 있도록 단계별로 정리했습니다.
앱 안에서도 동일한 내용을 **❓ 도움말** 탭에서 확인할 수 있습니다.

---

## 1. Trend Collector 란?

키워드 기반으로 언론 보도를 자동 수집·분석하여 리포트로 만들어주는 도구입니다.

- 네이버 / 구글 뉴스를 키워드별로 검색
- 본문 자동 추출 + 감정 분석 + 부서 분류
- **편철형(언론 스크랩철) / 분석형(보고서)** 두 종류로 출력
- 자동 수집 스케줄 + 메일 자동 발송 지원

> 본 도구는 내부 업무용 자료를 자동 생성합니다. 외부 공개 시에는 반드시 검토가 필요합니다.

---

## 2. 처음 사용하는 방법

1. **로그인** — 관리자가 발급한 비밀번호로 접속.
2. **🏷 키워드 탭** 으로 이동 — 빠른 키워드를 클릭하거나 직접 입력 후 Enter.
   처음에는 **1~3개** 정도만 넣는 것을 권장합니다.
3. **수집 기간** 선택 — 가장 안정적인 값은 **최근 7일** 또는 **최근 30일** 입니다.
4. **🔍 지금 즉시 수집** 버튼 클릭 — 15~30초 안에 리포트가 생성됩니다.
5. **📰 리포트 탭** 으로 이동 — 방금 만든 리포트를 열어 통계 / 본문 / 부정 이슈를 확인.
6. **출력 / 다운로드** — 리포트 상세 화면에서 **편철형** 또는 **분석형** 출력물을 PDF / Word / HTML / Excel 로 받을 수 있습니다.

---

## 3. 키워드 입력 / 빠른 키워드 사용법

### 빠른 키워드 (5 카테고리)

- 법무부(본부)
- 보호직(보호관찰 및 소년원) — *기본 펼침*
- 교정
- 출입국
- 검찰

각 카테고리는 **핵심 키워드** + **확장 키워드** 로 나뉘며, 핵심만 먼저 표시됩니다.
“▾ 확장 키워드 N개 — 더보기” 를 눌러 전체를 펼칠 수 있습니다.

### 추천 키워드

선택한 키워드와 자주 함께 사용되는 키워드가 자동으로 표시됩니다.
예) `보호관찰` 선택 → `보호관찰소 / 전자감독 / 전자발찌 / 사회봉사명령` 추천.

### 검색 목적 프리셋

“🎯 검색 목적” 에서 6 가지 목적 중 하나를 선택하면 추천 키워드 묶음이 표시됩니다.

- 일반 모니터링
- 부정 이슈 감지 — *논란 / 비판 / 의혹 / 구속 / 수사 / 부실 / 사고 / 피해 / 반발*
- 홍보성과 확인 — *보도자료 / 정책브리핑 / 성과 / 개선 / 지원 / 추진 / 수상 / 협약*
- 제도개선 이슈 발굴 — *제도개선 / 법령개정 / 사각지대 / 재범방지 / 인권보호*
- 국회·감사 대응 — *국회 / 국정감사 / 감사원 / 지적 / 질의 / 자료요구 / 예산*
- 사건·사고 모니터링 — *사고 / 사망 / 폭행 / 도주 / 적발 / 구속 / 피해 / 수사*

## 4. OR 검색과 AND 검색

**기본 검색은 OR 입니다.** 여러 키워드 중 **하나라도** 포함된 기사가 결과에 들어옵니다.

“모든 키워드를 포함하는 기사만” 체크박스를 켜면 **AND 검색** 으로 바뀝니다 — 매우 좁게 검색할 때만 사용하세요.

### 예시

키워드: `[보호관찰, 보호관찰소, 전자감독]`

- **OR (기본)**: 셋 중 하나라도 포함된 기사 → 결과 다수
- **AND (옵션 ON)**: 셋 모두 포함된 기사만 → 결과 적음 / 0건

> `보호관찰` 은 `보호관찰소` 안에 포함되므로, AND 시 자동으로 `보호관찰소` 만 검사합니다 (포함 관계 자동 축약).

## 5. 수집 기간 설정

| 옵션 | 추천 용도 |
|---|---|
| 최근 24시간 | 긴급 모니터링 (결과 적을 수 있음) |
| 최근 3일 / 7일 | 일일 / 주간 보고서 *(권장)* |
| 최근 14일 / 30일 | 월간 보고서 / 트렌드 분석 |
| 직접 설정 | 시작 / 종료 날짜 수동 입력 |

> 날짜를 추출하지 못한 기사는 “날짜 미확인” 으로 보존되어 결과에 포함됩니다.

## 6. 리포트 확인 방법

1. **📰 리포트 탭** 에서 가장 최근 리포트가 위에 표시됩니다.
2. 리포트 클릭 → 표지 / 통계 / 부정 이슈 / 본문이 표시됩니다.
3. 본문 추출 실패 기사는 **🔄 이 기사 재추출** 또는 원문 링크로 확인.
4. **✉️ 메일** 버튼으로 수신자에게 즉시 발송.

## 7. 편철형 vs 분석형 출력 차이

### 📰 편철형 출력물

- 실제 인쇄해서 넘겨보는 **언론 스크랩철** 형태
- 표지 → 언론사별 목차 → 언론사별 페이지 → (선택) 분석 부록
- 흑백 인쇄 최적화 / 명조 폰트 / A4 세로
- 출력 전 **표지 / 날짜 / 기관명** 등을 직접 수정 가능

### 📊 분석형 보고서

- 관리자 / 담당자가 읽기 위한 **보고자료**
- 1페이지 요약 → 종합 분석 → 주요 이슈 → 긍부정 → 홍보성과 → 대응 필요사항
- 편집 가능한 Word / Excel 제공
- 공공기관 보고 문체 (`~임` / `~함` / `~필요함`)

---

## 8. Word / Excel / PDF / HTML 차이

| 형식 | 용도 |
|---|---|
| **PDF** | 인쇄 / 보존 용. 편집 불가. |
| **Word(.docx)** | 편집 가능. 제목·본문·표를 수정해 최종 결재 자료로 정리. |
| **Excel(.xlsx)** | 분석 / 통계용. 언론사별 목차 · 부정 이슈 · 부서별 대응 등 시트. |
| **HTML** | 브라우저로 열어 **Ctrl+P** 로 인쇄 / PDF 저장. 오프라인 보관용. |

> PDF 생성에 실패하면 Word 또는 HTML 다운로드를 사용하세요. 실패 시 자동 안내가 표시됩니다.

## 9. 네이버 API 설정 방법

**관리자 한 명만 입력**하면 모든 직원이 PC / 스마트폰에서 공통으로 사용할 수 있습니다.

1. https://developers.naver.com/apps/#/list 접속 → 애플리케이션 등록 → 검색(뉴스) 권한 부여.
2. 발급된 Client ID / Secret 복사.
3. **🛠 설정 → 📰 뉴스 소스 설정** 에서 입력 후 저장.
4. 🏷 키워드 화면 하단의 **Naver News** 토글을 켭니다.

> Client Secret 은 화면에 다시 표시되지 않습니다. 변경하려면 새 값을 입력하세요.

## 10. 메일 설정 방법

메일 발송 방식은 4 가지 중 선택할 수 있습니다.

| 방식 | 특징 |
|---|---|
| **SMTP** | 네이버 / Gmail 등 일반 메일 서버 직접 연결 — Render 무료 플랜에서 차단될 수 있음 |
| **Resend API** | API 방식 — Render 무료 플랜에서도 정상 동작 |
| **SendGrid API** | API 방식 — Render 무료 플랜에서도 정상 동작 |
| **저장만 하기** | 실제 발송 안 함. 기능개선 제안 등은 서버에 저장됨 |

> 💡 Render **무료 플랜은 SMTP 포트(25 / 465 / 587)** 가 제한될 수 있습니다.
> 테스트 메일이 **Connection timeout** 으로 실패하면 메일 API 방식 또는 유료 플랜 전환을 권장합니다.
